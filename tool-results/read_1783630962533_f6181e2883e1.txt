   799в†’  1298в†’  1298в†’9.3.1 Repository Pattern en el frontend
   800в†’  1299в†’  1299в†’Todas las llamadas a Firebase desde el frontend se encapsulan detrГЎs de interfaces Repository. Por ejemplo,
   801в†’  1300в†’  1300в†’PostRepository expone mГ©todos como create(), list(), comment() y like() que internamente usan el SDK de

   803в†’  1302в†’  1302в†’otro backend sin reescribir componentes. Los repositorios se inyectan vГ­a React Context.
   804в†’  1303в†’  1303в†’9.3.2 Event-driven backend con triggers de Firestore
   805в†’  1304в†’  1304в†’La lГіgica reactiva se implementa mediante Cloud Functions reactivas a eventos de Firestore. Cuando un documento
   806в†’  1305в†’  1305в†’se crea o actualiza, Firestore emite un evento que dispara la funciГіn. Por ejemplo, onPostCreate se dispara al crear
   807в†’  1306в†’  1306в†’un documento en /posts, valida el contenido, otorga XP al autor y actualiza contadores. Este patrГіn desacopla la
   808в†’  1307в†’  1307в†’escritura de datos (frontend directo a Firestore) del enriquecimiento y validaciГіn de negocio (Cloud Functions).
   809в†’  1308в†’  1308в†’9.3.3 Serverless con escalado a cero
   810в†’  1309в†’  1309в†’Cloud Functions v2 escala automГЎticamente de 0 a N instancias segГєn la carga. El cold start es el principal desafГ­o;
   811в†’  1310в†’  1310в†’se mitiga configurando minInstances=1 en funciones crГ­ticas (auth, onboarding) y usando funciones mГЎs ligeras (sin
   812в†’  1311в†’  1311в†’dependencias pesadas) para triggers frecuentes. Las funciones HTTP se exponen detrГЎs de Firebase Hosting para
   813в†’  1312в†’  1312в†’aprovechar su CDN y reducir latencia.
   814в†’  1313в†’  1313в†’9.3.4 Single source of truth en Firestore Rules
   815в†’  1314в†’  1314в†’Las reglas de seguridad de Firestore son la Гєnica fuente de verdad para autorizaciГіn. El frontend puede ocultar
   816в†’  1315в†’  1315в†’botones o rutas segГєn el rol del usuario, pero esa lГіgica es cosmГ©tica; el backend (Firestore Rules) rechaza cualquier
   817в†’  1316в†’  1316в†’acceso no autorizado. Esta separaciГіn evita vulnerabilidades donde un usuario malicioso podrГ­a eludir la UI para
   818в†’  1317в†’  1317в†’acceder a datos.
